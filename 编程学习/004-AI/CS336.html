虚拟环境创建 ​

Conda ​

可以同时管理 Python 解释器版本和第三方库，隔离项目间的运行环境

sh
# 查看环境
conda env list
# 创建环境
conda create -n pytorch python=3.12
# 创建的同时安装软件包
conda create -n ai python=3.12 numpy pandas matplotlib
# 删除环境
conda remove -n pytorch --all
# 激活环境
conda activate pytorch
# 退出环境
conda deactivate
# 查看指定环境的软件包
conda list -n pytorch
# 安装软件包
# 当前环境安装
conda install numpy
# 指定环境安装
conda install -n pytorch scipy
# 升级软件包
conda update numpy
conda update --all
# 卸载软件包
conda remove numpy

张量tensors是计算基础

py
x = torch.ones(16, 32)
w = torch.ones(32, 2)
y = x @ w
assert y.size() == torch.Size([16, 2])
#这是基础的二维矩阵相乘，当维度更多时：
x = torch.ones(4, 8, 16, 32)
w = torch.ones(32, 2)
y = x @ w
assert y.size() == torch.Size([4, 8, 16, 2])
x是四维，前两个只代表了维度，实际还是后面的[16, 32] @ [32, 2]

当维度多起来后，这样的代码无法清晰表示转置后两维的意思，因此出现了jaxtyping和einops:

py
# Old way:
x = torch.ones(2, 2, 1, 3) # batch seq heads hidden

# New (jaxtyping) way:
x: Float[torch.Tensor, "batch seq heads hidden"] = torch.ones(2, 2, 1, 3)

# Note: 这只是文档形式的，没有强制类型检查

# Define two tensors:
x: Float[torch.Tensor, "batch seq1 hidden"] = torch.ones(2, 3, 4)

y: Float[torch.Tensor, "batch seq2 hidden"] = torch.ones(2, 3, 4)

# Old way:
z = x @ y.transpose(-2, -1)

# New (einops) way:
z = einsum(x, y, "batch seq1 hidden, batch seq2 hidden -> batch seq1 seq2")

# 或者使用 ... 代替前几个维度
z = einsum(x, y, "... seq1 hidden, ... seq2 hidden -> ... seq1 seq2")
